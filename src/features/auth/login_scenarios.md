# Discord 登入狀態與異常處理指南

本文件記錄了 Kazran Alliance System 中，使用者透過 Discord 登入時可能遇到的各種情境，以及系統對應的處理方式與畫面提示。

## 1. 正常登入流程
當使用者符合以下所有條件時，將能順利登入系統：
1. Discord 帳號已完成 Email 或手機認證。
2. 已經加入指定的聯盟 Discord 伺服器 (Guild)。
3. 在該伺服器中擁有棕二相關的身份組。

系統會自動同步使用者的 Discord 暱稱、頭像與公會身分，並進入系統主畫面。

---

## 2. 異常登入情境與系統提示

為了確保系統安全與資料正確性，系統會在登入流程的各個階段進行檢查。以下是常見的異常情境與對應的提示：

### 情境 A：Discord 帳號未認證
* **條件**：登入者的 Discord 帳號尚未完成 Email 或手機號碼的認證。
* **發生階段**：在點擊「Discord 登入」後，跳轉至 Discord 官方授權頁面時。
* **系統提示**：Discord 官方頁面會直接阻擋授權，並顯示以下英文錯誤訊息：
  > **"You need a verified e-mail or phone number in order to perform this action."**
* **處理方式**：使用者必須先前往 Discord 設定中完成帳號認證，才能繼續授權流程。

### 情境 B：使用者不在聯盟 Discord 伺服器內
* **條件**：登入者雖然成功授權，但其 Discord 帳號並未加入系統指定的聯盟 Discord 伺服器。
* **發生階段**：授權完成並導回系統後，前端嘗試同步資料時。
* **系統提示**：系統會強制將該使用者登出，並在畫面右下角（或上方）顯示紅色的 Toast 提示：
  > **「登入失敗：您不在指定的公會聯盟中」**
* **後台紀錄**：系統會自動在 `system_logs` 中記錄一筆 `level: 'warn'`，`action: 'unauthorized_login'` 的日誌，方便管理員追蹤未授權的登入嘗試。

### 情境 C：使用者在伺服器內，但缺少「棕二」身份組
* **條件**：登入者已經加入聯盟 Discord 伺服器，但在伺服器中尚未被賦予任何包含「公會成員-棕色2」字樣的身份組。
* **發生階段**：授權完成並導回系統後，前端成功讀取到 Profile，但發現其 `user_guilds` 為空。
* **系統提示**：系統會彈出一個 Modal 視窗，提示使用者：
  > **「身份綁定出現問題」**
  （通常會引導使用者前往 Discord 伺服器的特定頻道申請身份組，或聯絡管理員處理。）
  系統會在 `profiles` 中記錄一筆 `id`、`user_role`、`user_guilds`為null的資料，並同步他的 `discord_id`和`auth_id`
* **處理方式**：使用者需要在 Discord 中取得正確的身份組後，重新登入系統以觸發資料同步。

## 3. 開發與除錯建議
* **無痕模式測試**：在測試不同情境時，強烈建議使用瀏覽器的「無痕模式 (Incognito)」，以避免舊的 Session 或 Local Storage 殘留導致測試結果不準確。
* **查看系統日誌**：若有使用者反映無法登入，管理員可前往「Admin Panel > 系統日誌 (System Logs)」查看是否有相關的錯誤紀錄（如 `unauthorized_login` 或 `sync_discord_roles` 的錯誤）。
